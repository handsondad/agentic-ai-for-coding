## 核心模块说明

### src/agent/ — Agent 核心

**职责**：Agent 的主要业务逻辑，包括：
- **core.py**：Agent 主循环，处理用户输入，协调工具调用和记忆检索
- **session.py**：会话生命周期管理（创建、更新、终止）

**设计决策**：
- 使用状态机管理会话状态（`idle → processing → waiting_tool → done`）
- 对话历史使用滑动窗口截断，保留最近 N 条消息

### src/tools/ — 工具注册与执行

**职责**：定义和管理 Agent 可以调用的工具（函数）

**设计模式**：插件式注册
```python
# 注册工具示例
from src.tools.registry import tool_registry

@tool_registry.register(
    name="search_web",
    description="搜索网络获取最新信息",
    parameters=SearchParams
)
async def search_web(params: SearchParams) -> SearchResult:
    ...
```

### src/memory/ — 记忆管理

**职责**：管理 Agent 的短期和长期记忆
- **短期记忆**：当前会话的对话历史（内存存储）
- **长期记忆**：跨会话的用户信息和知识（持久化存储）

### src/api/ — HTTP API 层

**职责**：对外暴露 RESTful API 和 WebSocket 接口
- **routes.py**：API 路由定义
- 认证、请求验证、限流在中间件中处理

## 数据流

### 标准请求处理流程

```
用户请求
  │
  ▼
API 层（认证 + 参数验证）
  │
  ▼
Agent 核心（加载会话 + 检索记忆）
  │
  ▼
LLM 调用（构建 Prompt + 发送请求）
  │
  ├── 需要工具调用 ──▶ 工具层（执行工具）──▶ 返回结果给 LLM
  │
  ▼
更新记忆（保存对话历史）
  │
  ▼
返回响应给用户
```

## 非功能性设计

### 性能
- 所有 LLM 调用使用流式响应，减少首字节延迟
- 使用连接池管理数据库连接
- 目标：P99 延迟 < 2 秒（不含 LLM 响应时间）

### 可靠性
- LLM API 调用使用指数退避重试（最多 3 次）
- 关键操作幂等设计，支持安全重试
- 目标可用性：99.9%

### 安全性
- 所有 API 端点需要认证（JWT）
- 用户数据隔离（多租户场景）
- 工具调用白名单控制
- 敏感数据加密存储

### 可观测性
- 结构化日志（JSON 格式）
- 关键操作的 Trace ID 追踪
- 核心指标上报（请求量、延迟、错误率、LLM Token 消耗）

## 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| Web 框架 | FastAPI | 原生异步支持，自动 API 文档 |
| LLM 客户端 | openai Python SDK | 官方支持，功能完整 |
| 数据验证 | Pydantic v2 | 类型安全，性能好 |
| 测试框架 | pytest + pytest-asyncio | 生态成熟，异步支持好 |
| 代码质量 | ruff | 速度快，规则全 |
